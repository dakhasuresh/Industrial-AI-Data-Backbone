# Contributing to Industrial-AI-Data-Backbone

Thank you for your interest in contributing to this reference architecture repository. Contributions from Industrial AI practitioners, OT architects, and domain experts are welcome.

---

## What We're Looking For

This is a **reference architecture repository** — not a software project. Contributions should focus on:

- **Architecture diagrams** — new or improved Mermaid diagrams illustrating industrial AI patterns
- **Framework documentation** — additions or refinements to architecture guidance
- **Use cases** — real-world industrial AI use cases from manufacturing, utilities, energy, oil & gas, or mining
- **Templates** — reusable assessment, roadmap, or design templates
- **Corrections** — factual corrections to architecture guidance or standards references

---

## How to Contribute

1. **Fork** this repository
2. **Create a branch** for your contribution: `git checkout -b feature/your-contribution-name`
3. **Make your changes** following the style guide below
4. **Submit a Pull Request** with a clear description of what you've added or changed

---

## Style Guide

### Markdown

- Use `##` for top-level sections, `###` for sub-sections
- Use tables for structured comparisons
- Reference other files using relative links: `[Guide](/docs/unified-namespace-guide.md)`
- Include a brief description at the top of each file explaining its purpose and audience

### Mermaid Diagrams

- Use `graph TB` (top-to-bottom) for architecture diagrams
- Use `sequenceDiagram` for process flows and integration patterns
- Use `gantt` for roadmaps and timelines
- Apply `classDef` styling for visual consistency
- Test diagrams render correctly before submitting (use the [Mermaid Live Editor](https://mermaid.live))

### Language & Tone

- Write for a senior technical and executive audience
- Be specific — avoid vague generalisations
- Cite standards (IEC 62443, ISA-95, ISA-88) correctly
- Use British English spelling as the primary standard (e.g., "contextualisation", "organisation")

---

## What We Don't Accept

- Software code (this is not a software repository)
- Vendor-specific tutorials or product documentation
- Content that promotes specific vendors without architectural justification
- Speculative content not grounded in industrial practice

---

## Reporting Issues

If you find inaccuracies, outdated references, or missing content, please open a GitHub Issue with:

- The file and section affected
- A description of the issue
- Suggested correction (if known)
